# Contributing to Project Odyssey

Thank you for your interest in Project Odyssey.

This repository documents my homelab environment and serves as a learning project focused on Linux, Docker, networking, automation, infrastructure, and DevOps practices.

## Project Goals

* Build a production-style homelab.
* Learn infrastructure through practical experience.
* Maintain clear documentation.
* Follow reproducible deployment practices.

## Repository Structure

* `compose/` – Docker Compose configurations and examples.
* `docs/` – Documentation, architecture, lessons learned, and operational procedures.
* `scripts/` – Helper scripts and automation.
* `README.md` – Project overview.

## Contribution Guidelines

When making changes:

* Keep documentation up to date.
* Test Docker Compose changes before committing.
* Never commit passwords, secrets, or `.env` files.
* Use descriptive commit messages.
* Keep scripts well commented.
* Maintain consistent formatting across documentation.

## Reporting Issues

If you discover an issue, please open an issue describing:

* What happened.
* Expected behaviour.
* Steps to reproduce.
* Relevant logs (where appropriate).

## Code Style

* Prefer simplicity over complexity.
* Document decisions where appropriate.
* Keep the repository beginner-friendly and educational.

Thank you for helping improve Project Odyssey.
